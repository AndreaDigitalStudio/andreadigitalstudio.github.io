## Para qué sirve

Un tag marca un commit concreto como una versión importante del proyecto — por ejemplo `v1.0`, `v2.1.3`. A diferencia de las ramas, un tag no se mueve: siempre apunta al mismo commit, sirviendo como un punto de referencia fijo en el historial.

## Tipos de tags

| Tipo | Para qué |
| --- | --- |
| Ligero (lightweight) | Solo un nombre apuntando a un commit |
| Anotado (annotated) | Incluye autor, fecha, mensaje y puede firmarse — recomendado para releases |

## Crear un tag ligero

```bash
git tag v1.0
```

Marca el commit actual (`HEAD`) con el tag `v1.0`.

## Crear un tag anotado (recomendado)

```bash
git tag -a v1.0 -m "Primera versión estable"
```

Guarda información adicional: quién lo creó, cuándo y el mensaje.

## Etiquetar un commit anterior

```bash
git log --oneline
```

```text
a1b2c3d feat: añadir página de categoría Git
e4f5g6h fix: corregir ruta del CSS
```

```bash
git tag -a v0.9 e4f5g6h -m "Versión previa al lanzamiento"
```

## Ver los tags existentes

```bash
git tag
```

```text
v0.9
v1.0
```

## Ver información de un tag

```bash
git show v1.0
```

Muestra el mensaje del tag y el commit al que apunta.

## Subir tags a GitHub

Los tags **no se suben automáticamente** con `git push`.

Subir un tag concreto:

```bash
git push origin v1.0
```

Subir todos los tags:

```bash
git push origin --tags
```

## Borrar un tag

Local:

```bash
git tag -d v1.0
```

Remoto (GitHub):

```bash
git push origin --delete v1.0
```

## Volver a un tag concreto

```bash
git checkout v1.0
```

> Esto te deja en estado "detached HEAD" — útil para revisar cómo era el proyecto en esa versión, pero no para seguir desarrollando ahí. Para trabajar a partir de un tag, crear una rama nueva:
> ```bash
> git checkout -b hotfix-v1.0 v1.0
> ```

## Relación con GitHub Releases

En GitHub, un **Release** se basa en un tag — al crear un Release desde la web, GitHub permite elegir un tag existente o crear uno nuevo, y añadir notas de versión, binarios adjuntos, etc.

## Resumen

| Acción | Comando |
| --- | --- |
| Crear tag anotado | `git tag -a v1.0 -m "mensaje"` |
| Ver tags | `git tag` |
| Ver detalles de un tag | `git show v1.0` |
| Subir un tag | `git push origin v1.0` |
| Subir todos los tags | `git push origin --tags` |
| Borrar tag local | `git tag -d v1.0` |
| Borrar tag remoto | `git push origin --delete v1.0` |
