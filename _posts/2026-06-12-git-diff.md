## Para qué sirve

`git diff` muestra las diferencias línea por línea entre distintos estados del proyecto — útil para revisar exactamente qué has cambiado antes de hacer `git add` o `git commit`, y evitar subir algo por error.

## Cómo se lee

```diff
- línea eliminada (en rojo)
+ línea añadida (en verde)
```

Ejemplo:

```diff
- const puerto = 3000;
+ const puerto = 8080;
```

Significa que esa línea cambió de `3000` a `8080`.

## Comandos básicos

### Cambios sin confirmar (sin `git add`)

```bash
git diff
```

Muestra las diferencias entre tu directorio de trabajo y el último commit.

### Cambios ya añadidos al staging

```bash
git diff --staged
```

(o `--cached`, es lo mismo). Muestra lo que se incluiría si hicieras `git commit` ahora.

### Diferencias de un archivo concreto

```bash
git diff archivo.txt
git diff --staged archivo.txt
```

## Comparar entre commits

```bash
git diff <commit1> <commit2>
```

Ejemplo:

```bash
git diff HEAD~1 HEAD
```

Muestra qué cambió en el último commit.

## Comparar entre ramas

```bash
git diff main feature/menu-responsive
```

Muestra las diferencias entre `main` y la rama indicada.

## Resumen visual del flujo

```text
Directorio de trabajo  →  git diff           →  Staging area
Staging area           →  git diff --staged  →  Último commit
```

## Resumen

| Quiero ver... | Comando |
| --- | --- |
| Cambios sin `git add` | `git diff` |
| Cambios ya añadidos (staged) | `git diff --staged` |
| Diferencias de un archivo | `git diff archivo.txt` |
| Diferencias entre dos commits | `git diff <commit1> <commit2>` |
| Diferencias entre ramas | `git diff main otra-rama` |
